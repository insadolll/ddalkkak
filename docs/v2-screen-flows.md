## 시나리오 1: 신규 프로젝트 Full Cycle

담당자가 프로젝트를 수립하고, 견적서 교환, 수주 확정, 계산서까지 완료하는 전체 흐름.

```
[대시보드]
    │
    │ 사이드바 "프로젝트 관리" 클릭
    ▼
[프로젝트 목록]
    │
    │ [+ 새 프로젝트] 클릭
    ▼
[프로젝트 생성 폼] ─────────────────────────────────────────────────────
    │
    │ 프로젝트명, 고객사, 매입처, 담당자, 기간 입력
    │ [저장] 클릭
    │
    │ ✓ 토스트: "프로젝트가 생성되었습니다"
    ▼
[프로젝트 상세] ─── 타임라인: ◉ 수립 ─ ○ ─ ○ ─ ○ ─ ○ ─ ○ ──────────
    │
    │ ── 매입 견적서 등록 ──
    │ 매입 견적서 풀 아코디언 열기
    │ [+ 견적 등록] 클릭
    ▼
[매입 견적서 등록 모달] ─────────────────────────────────────────────────
    │
    │ 매입처 확인, 원본 PDF 업로드, 품목/수량/단가 입력
    │ [등록] 클릭
    │
    │ ✓ 토스트: "매입 견적서가 등록되었습니다"
    │ ✓ 매입 풀에 Mini 카드 추가 (fade-in)
    ▼
[프로젝트 상세] ─── 타임라인: ● 수립 ─ ◉ 견적수령 ─ ○ ─ ○ ─ ○ ─ ○ ─
    │
    │ ── 매출 견적서 자동 생성 ──
    │ 매입 카드의 [⋮] → "매출 견적서 생성" 클릭
    ▼
[마진 설정 모달] ────────────────────────────────────────────────────────
    │
    │ 마진율 or 마진액 설정
    │ [생성] 클릭
    ▼
[견적서 편집 페이지] ── /quotations/new?from=purchase&id=xxx ───────────
    │
    │ 품목 자동 채워짐 (마진 반영된 단가)
    │ 필요 시 품목 수정/추가/삭제
    │ [저장] 클릭
    │
    │ ✓ 토스트: "매출 견적서가 생성되었습니다"
    │ ✓ 자동으로 프로젝트의 매출 풀에 연결
    ▼
[프로젝트 상세] ─── 매출 풀에 새 카드 추가됨 ────────────────────────────
    │
    │ ── 견적서 메일 발송 ──
    │ 매출 견적서 카드의 [📧 메일] 클릭
    ▼
[메일 발송 확인 모달] ──────────────────────────────────────────────────
    │
    │ 수신자 확인/수정, 제목 확인, PDF 첨부 미리보기
    │ [발송] 클릭
    │
    │ ✓ 토스트: "견적서가 메일로 발송되었습니다"
    │ ✓ 카드 상태: 작성중 → 발송완료
    ▼
[프로젝트 상세] ─── 타임라인: ● ─ ● ─ ◉ 견적발송 ─ ○ ─ ○ ─ ○ ──────
    │
    │ ── (고객사와 네고, 리비전 반복 가능) ──
    │ 매출 카드 [⋮] → "새 리비전"
    │ → 견적서 편집 (Rev.1, Rev.2...)
    │ → 메일 재발송
    │
    │ ── 견적서 확정 ──
    │ 최종 매입 카드 [⋮] → "확정"
    │ 최종 매출 카드 [⋮] → "확정"
    ▼
[확정 확인 모달] ───────────────────────────────────────────────────────
    │
    │ "이 견적서를 확정하시겠습니까?"
    │ "확정 시 매출/매입 금액이 프로젝트에 반영됩니다"
    │ [확정] 클릭
    │
    │ ✓ 토스트: "견적서가 확정되었습니다"
    │ ✓ 카드: ★ 확정 뱃지 + 테두리 강조
    │ ✓ 요약 패널: 매출/매입/수익 금액 자동 갱신
    │ ✓ 캘린더: 결제 마감일 자동 등록 (결제조건 기반)
    ▼
[프로젝트 상세] ─── 타임라인: ● ─ ● ─ ● ─ ◉ 수주확정 ─ ○ ─ ○ ──────
    │
    │ ── 발주서 생성 ──
    │ 확정된 매입 카드 [⋮] → "발주서로 변환"
    ▼
[발주서 변환 확인 모달] ────────────────────────────────────────────────
    │
    │ "견적서를 발주서로 변환합니다"
    │ "문서 제목과 문구가 자동으로 변경됩니다"
    │ 미리보기 (변경 전 → 후 비교)
    │ [변환] 클릭
    │
    │ ✓ 토스트: "발주서가 생성되었습니다"
    │ ✓ 발주서 카드 생성 (보라색 뱃지)
    ▼
[프로젝트 상세] ─── 타임라인: ● ─ ● ─ ● ─ ● ─ ◉ 납품 ─ ○ ──────────
    │
    │ ── 납품 완료 후 계산서 요청 ──
    │ 계산서 아코디언 열기
    │ [계산서 요청 메일 보내기] 클릭
    ▼
[계산서 요청 모달] ─────────────────────────────────────────────────────
    │
    │ 수신: 경영지원 담당자
    │ 내용: 프로젝트 정보 링크 포함
    │ [발송] 클릭
    │
    │ ✓ 토스트: "계산서 요청 메일이 발송되었습니다"
    │ ✓ 계산서 상태: 미발행 → 요청중
    ▼
[프로젝트 상세] ─── 타임라인: ● ─ ● ─ ● ─ ● ─ ● ─ ◉ 계산서 ────────
    │
    │ 경영지원 담당자가 계산서 처리 후 상태 변경
    │
    │ ✓ 모든 단계 완료
    ▼
[프로젝트 상세] ─── 타임라인: ● ─ ● ─ ● ─ ● ─ ● ─ ● (완료) ────────
```

## 시나리오 2: 매입 견적서 수동 등록

거래처에서 메일/팩스로 받은 견적서를 직접 등록.

```
[프로젝트 상세]
    │
    │ 매입 견적서 풀 → [+ 견적 등록] 클릭
    ▼
[매입 견적서 등록 모달] ────────────────────────────────────────────────
    │
    │  ┌──────────────────────────────────────────┐
    │  │ 매입 견적서 등록                           │
    │  │                                           │
    │  │ 매입처:  [B사 ▾]  (자동 or 검색)          │
    │  │                                           │
    │  │ 원본 파일:                                 │
    │  │ ┌─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┐  │
    │  │ │  📎 파일을 끌어다 놓거나 클릭       │  │
    │  │ │     PDF, Excel, 이미지              │  │
    │  │ └─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┘  │
    │  │                                           │
    │  │ 품목 입력:                                 │
    │  │ ┌────┬──────┬─────┬────────┬──────────┐  │
    │  │ │ #  │ 품명  │ 수량 │ 단가    │ 금액      │  │
    │  │ ├────┼──────┼─────┼────────┼──────────┤  │
    │  │ │ 1  │      │     │        │ 자동계산  │  │
    │  │ └────┴──────┴─────┴────────┴──────────┘  │
    │  │                        [+ 품목 추가]      │
    │  │                                           │
    │  │ 공급가액: ₩0            세액: ₩0          │
    │  │                                           │
    │  │ 비고: [                        ]           │
    │  │                                           │
    │  │              [취소]  [등록]                │
    │  └──────────────────────────────────────────┘
    │
    │ [등록] 클릭
    │ ✓ 매입 풀에 Mini 카드 추가
    │ ✓ 원본 PDF 첨부문서에 자동 보관
    ▼
[프로젝트 상세] ─── 매입 풀에 새 카드 나타남 ─────────────────────────
```

## 시나리오 3: 매입→매출 견적서 자동 생성

매입 견적서를 기반으로 마진 반영한 매출 견적서 자동 생성.

```
[프로젝트 상세] ─── 매입 풀의 카드 ────────────────────────────────────
    │
    │ 매입 카드 hover → [⋮] → "매출 견적서 생성" 클릭
    ▼
[마진 설정 모달] ───────────────────────────────────────────────────────
    │
    │  ┌─────────────────────────────────────┐
    │  │ 매출 견적서 자동 생성                 │
    │  │                                      │
    │  │ 기준 견적서: B사 견적 v2              │
    │  │ 매입가:      ₩11,200,000             │
    │  │                                      │
    │  │ ── 마진 설정 ──                      │
    │  │                                      │
    │  │ ○ 마진율  [     ]%                   │
    │  │ ● 마진액  [3,800,000] 원             │
    │  │                                      │
    │  │ 예상 매출가: ₩15,000,000             │
    │  │ 예상 마진율: 25.3%                   │
    │  │                                      │
    │  │ ── 적용 방식 ──                      │
    │  │                                      │
    │  │ ● 전체 균등 (모든 품목 동일 비율)     │
    │  │ ○ 품목별 조정 (다음 단계)            │
    │  │                                      │
    │  │          [취소]  [다음 →]             │
    │  └─────────────────────────────────────┘
    │
    │ "전체 균등" + [다음] 클릭
    ▼
[견적서 편집 페이지] ── 자동 채워진 상태 ───────────────────────────────
    │
    │ 품목 목록이 매입 견적서 기반 + 마진 반영된 단가로 채워짐
    │ 상단에 안내 배너:
    │ ┌──────────────────────────────────────────────────────┐
    │ │ ℹ B사 견적 v2 기반으로 자동 생성되었습니다.           │
    │ │   마진율 25.3% 적용. 품목별 수정이 가능합니다.        │
    │ └──────────────────────────────────────────────────────┘
    │
    │ 제목, 수신자, 품목 세부 조정
    │ [저장] 클릭
    │
    │ ✓ 토스트: "매출 견적서가 생성되었습니다"
    ▼
[프로젝트 상세] ─── 매출 풀에 새 카드 (출처 링크 포함) ────────────────

    ┌────────────────────────────────┐
    │  Q-2026-001             Rev.0  │
    │  A사 견적서                     │
    │  ₩15,000,000                   │
    │  마진 25.3%                    │  ← 매입 대비 마진 표시
    │  ↑ B사 견적 v2 기반             │  ← 출처 링크
    └────────────────────────────────┘
```

## 시나리오 4: 견적서 메일 발송 (딸깍)

클릭 한 번으로 견적서 PDF를 메일 발송.

```
[프로젝트 상세 or 견적서 상세]
    │
    │ 카드/페이지에서 [📧 메일발송] 클릭
    ▼
[메일 발송 모달] ──────────────────────────────────────────────────────
    │
    │  ┌────────────────────────────────────────────────┐
    │  │ 견적서 메일 발송                                │
    │  │                                                │
    │  │ 발신자:  홍길동 <hong@hubiocem.com> [변경 ▾]   │
    │  │ 수신자:  김부장 <kim@a-company.com> [+ 추가]   │
    │  │ 참조:    [                           ] [+ 추가]│
    │  │                                                │
    │  │ 제목:                                          │
    │  │ [견적서 송부의 건 - Q-2026-001 (Rev.2)]        │
    │  │                                                │
    │  │ 본문:                                          │
    │  │ ┌──────────────────────────────────────────┐   │
    │  │ │ 안녕하세요, 휴바이오켐 홍길동입니다.       │   │
    │  │ │                                          │   │
    │  │ │ 요청하신 견적서를 송부드립니다.            │   │
    │  │ │ 검토 후 회신 부탁드립니다.                 │   │
    │  │ │                                          │   │
    │  │ │ 감사합니다.                               │   │
    │  │ └──────────────────────────────────────────┘   │
    │  │                                                │
    │  │ 첨부:                                          │
    │  │ 📎 Q-2026-001-R2_A사_견적서.pdf  (자동 생성)  │
    │  │ [+ 추가 파일 첨부]                              │
    │  │                                                │
    │  │ ┌──────────────────────────────────────────┐   │
    │  │ │ [PDF 미리보기 썸네일]                     │   │
    │  │ │                                          │   │
    │  │ │  클릭하여 미리보기                        │   │
    │  │ └──────────────────────────────────────────┘   │
    │  │                                                │
    │  │               [취소]  [📧 발송]                │
    │  └────────────────────────────────────────────────┘
    │
    │ [발송] 클릭
    │
    │ ✓ 버튼: 스피너 → "발송 완료 ✓" (1.5초) → 모달 닫힘
    │ ✓ 토스트: "견적서가 kim@a-company.com으로 발송되었습니다"
    │ ✓ 카드 상태: 발송완료 뱃지로 변경
    │ ✓ 히스토리에 발송 기록 추가
    ▼
[프로젝트 상세] ─── 카드 상태 갱신됨 ──────────────────────────────────
```

## 시나리오 5: 견적서 → 발주서 변환

확정된 매입 견적서를 우리 양식 발주서로 변환.

```
[프로젝트 상세] ─── 확정된 매입 카드 ──────────────────────────────────
    │
    │ 매입 카드 [⋮] → "발주서로 변환" 클릭
    ▼
[발주서 변환 모달] ─────────────────────────────────────────────────────
    │
    │  ┌────────────────────────────────────────────────┐
    │  │ 견적서 → 발주서 변환                            │
    │  │                                                │
    │  │ 기준: B사 견적 v2 (★ 확정)                     │
    │  │                                                │
    │  │ ── 자동 변경 사항 ──                            │
    │  │                                                │
    │  │  변경 전              변경 후                   │
    │  │  ┌──────────┐       ┌──────────┐              │
    │  │  │ 견 적 서  │  →    │ 발 주 서  │              │
    │  │  │          │       │          │              │
    │  │  │ "귀사에   │       │ "아래와   │              │
    │  │  │  아래와   │       │  같이     │              │
    │  │  │  같이     │       │  발주     │              │
    │  │  │  견적..." │       │  합니..." │              │
    │  │  └──────────┘       └──────────┘              │
    │  │                                                │
    │  │ ── 발주 정보 추가 ──                            │
    │  │                                                │
    │  │ 납품 요청일: [2026-04-15]                       │
    │  │ 납품 장소:   [                          ]       │
    │  │                                                │
    │  │                [취소]  [발주서 생성]             │
```

| 마진 설정 | 매입 카드 → 매출 견적서 생성 | `max-w-md` |
| 메일 발송 확인 | 카드/상세 [메일발송] | `max-w-xl` |
| 발주서 변환 | 확정 카드 → 발주서 변환 | `max-w-lg` |
| 확정 확인 | 카드 → 확정 | `max-w-sm` (confirm) |
| 계산서 요청 | 프로젝트 상세 [계산서 요청] | `max-w-lg` |
| 단건 거래 등록 | 매입매출 [+ 단건 등록] | `max-w-md` |
| 품목 입력 (IMAP) | 수신 카드 [품목 입력] | `max-w-4xl` (PDF+폼 2열) |
| 엑셀 업로드 | 프로젝트 목록 [엑셀 업로드] | `max-w-2xl` (스텝) |

### 모달 공통 규칙

- 배경: `bg-black/40 backdrop-blur-sm`
- 진입: `scale(0.95) opacity-0 → scale(1) opacity-1` (200ms ease-out)
- 닫기: 우상단 X 버튼, ESC 키, 배경 클릭 (폼 모달은 배경 클릭 비활성)
- 폼 모달: 변경사항 있을 때 닫기 시 "저장하지 않고 닫으시겠습니까?" 확인
- 모바일: 하단에서 올라오는 시트 (bottom sheet) 스타일
